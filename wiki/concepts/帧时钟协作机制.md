# 帧时钟协作机制

## 定义

CMP 中用于在 VSync 周期内按正确时序调度 Recomposition、Animation/Effect、Layout/Draw 三类工作的协程调度系统。

## 详解

### 两级 BroadcastFrameClock 级联

系统中存在两个独立的 BroadcastFrameClock：

1. **BaseComposeScene.frameClock**：场景级帧时钟，sendFrame 由 render() 调用，只在有 VSync 时触发
2. **Recomposer.broadcastFrameClock**：用户级帧时钟，由 recomposer 在帧内调用，可决定是否传播

### render() 六阶段

| 阶段 | 操作 | 说明 |
|------|------|------|
| 1 | flush effectDispatcher | 处理上帧产出的 effect 任务（动画、LaunchedEffect） |
| 2 | flush recomposeDispatcher | 处理 recomposer continuation resume |
| 3 | frameClock.sendFrame | 级联触发：recomposer → effect onFrame → recompose → applyChanges |
| 4 | measure + layout | 使用阶段 3 更新后的 Node 树 |
| 5 | pointer + Snapshot 通知 | 合成通知 |
| 6 | draw | 绘制到 Canvas |

### FlushCoroutineDispatcher 双路径

- **路径 A**（帧内）：immediateTasks 同步队列，flush() 消费
- **路径 B**（帧间）：scope.launch via OhosUiDispatcher，确保无 render() 时任务仍能执行

### 关键约束

- recomposeDispatcher 最多 1 个任务（单协程 + 单 continuation）
- effectDispatcher 可有 N 个任务（多协程）
- sendFrame 内所有 onFrame 同步执行
- LaunchedEffect 有一帧启动延迟

## 关键要点

- 阶段 1-2 处理上帧产出，阶段 3 产出本帧任务给下一帧
- 路径 B 是"自举"路径：状态变化驱动 recomposer 到达 withFrameNanos → onNewAwaiters → 帧请求 → render()
- OhosUiDispatcher 的 MonotonicFrameClock 在 CMP 中无直接调用者
