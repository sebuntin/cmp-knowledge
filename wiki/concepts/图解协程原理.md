# 图解协程原理

## 定义

Kotlin 协程通过编译器 CPS（Continuation-Passing-Style）变换将 suspend 函数转为状态机，实现轻量级用户态异步并发。

## 详解

### CPS 变换

编译器为每个 suspend 函数追加 `Continuation` 参数：

```
suspend fun foo(): String
  编译后 →
fun foo(continuation: Continuation<String>): Any?
  // 返回值：COROUTINE_SUSPENDED | 实际结果 | null
```

### 状态机实现

编译器生成 `ContinuationImpl` 子类，核心结构：

```kotlin
class FooContinuation(completion: Continuation<*>) : ContinuationImpl(completion) {
    var label: Int = 0        // 当前状态
    var result: Any? = null   // 中间结果

    override fun invokeSuspend(result: Any?): Any? {
        this.result = result
        when (label) {
            0 -> { label = 1; suspendFun1(this); }  // 状态0 → 挂起点1
            1 -> { label = 2; suspendFun2(this); }  // 状态1 → 挂起点2
            2 -> { return finalResult }               // 状态2 → 完成
        }
        return COROUTINE_SUSPENDED
    }
}
```

单实例复用：整个协程只创建一个 Continuation 对象，所有状态转换共享。

### 协程 vs 线程

| 维度 | 线程 | 协程 |
|------|------|------|
| 调度 | 内核抢占式 | 用户态协作式 |
| 开销 | 栈 MB 级 | Continuation 对象 KB 级 |
| 切换 | 系统调用 | 状态机跳转 |
| 数量 | 受限于内存 | 可创建数十万 |

## 关键要点

- suspend 函数的本质是带 Continuation 参数的普通函数
- 状态机通过 label 整数 + when 块实现状态转换
- 单个 Continuation 实例跨状态复用，内存高效
- 协程是用户态协作式调度，通过 Dispatcher 切换线程
