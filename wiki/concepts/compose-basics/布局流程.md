# 布局流程

## 定义

Compose 布局流程由 MeasurePolicy 定义，将测量+放置合并为单次调用，Constraints 自顶向下传播，强制单次测量保证 O(n) 布局时间，Intrinsic Measurement 解决单次测量下的尺寸协商。

## 详解

### 三步合一

```
MeasureScope.measure(measurables, constraints) {
    // 步骤1: 测量子节点
    val placeables = measurables.map { it.measure(childConstraints) }

    // 步骤2: 决定自身尺寸
    val width = maxOf(placeables.map { it.width })
    val height = placeables.sumOf { it.height }

    // 步骤3: 放置子节点
    layout(width, height) {
        var y = 0
        placeables.forEach { placeable ->
            placeable.place(0, y)
            y += placeable.height
        }
    }
}
```

Kotlin 类型系统强制顺序：`measure` 返回 `Placeable` → `place` 需要 `Placeable` → `layout()` 必须是最终返回值。

### Constraints 传播

```
根节点 Constraints(0..1080, 0..1920)
  → Column 约束子节点最大宽度 1080
    → Row 约束子节点各占一半 0..540
      → Text 在 540 宽内测量自身
```

比 Android MeasureSpec 更显式，避免多轮测量。

### 单次测量保证

同一 Measurable 测量两次抛异常：

```kotlin
val size = measurable.measure(Constraints())  // OK
val size2 = measurable.measure(Constraints()) // 💥 IllegalStateException!
```

保证 O(n) 布局时间（n = 节点数），不论嵌套深度。

### Intrinsic Measurement

正式测量前的预查询，解决"先有鸡还是先有蛋"问题：

```kotlin
Row {
    Text("Hello")       // 子节点1
    Divider()           // 子节点2，高度应与最高的兄弟对齐
    Text("World!")      // 子节点3
}.height(IntrinsicSize.Min)  // 使用固有高度
```

| 方法 | 含义 |
|------|------|
| `minIntrinsicWidth(height)` | 给定高度下最小宽度 |
| `maxIntrinsicWidth(height)` | 给定高度下最大宽度 |
| `minIntrinsicHeight(width)` | 给定宽度下最小高度 |
| `maxIntrinsicHeight(width)` | 给定宽度下最大高度 |

## 关键要点

- 布局三步合一：测子节点 → 定自身尺寸 → 放置子节点
- MeasurePolicy 是函数式接口，每个布局容器提供自己的实现
- 强制单次测量保证 O(n) 布局时间
- Intrinsic Measurement 是单次测量下的尺寸协商机制
- Modifier 节点在布局树中形成拦截约束的子链
