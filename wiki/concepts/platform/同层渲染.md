# 同层渲染

## 定义

OHOS FusionRenderer 路径中，通过 `ExternalRenderNode` 将 ArkUI 原生组件直接嵌入 Compose 绘制管线的混排方式。原生组件与 Compose 内容在同一渲染层合成，无需叠层或挖洞。

## 详解

### 架构原理

同层渲染的核心是将 ArkUI 的 RenderNode 通过 `ExternalRenderNode` 桥接对象注入 Compose 的绘制遍历。Compose 的 `DrawModifier` 在 `onDraw()` 中调用 `canvas.nativeCanvas.drawRenderNode(externalRenderNode, rect)`，将原生组件的渲染结果合成为 Compose 画面的一部分。

```
Compose 绘制遍历
  → DrawModifier.onDraw()
    → SkCanvas.drawRenderNode(externalRenderNode, rect)
      → OH_Drawing 回放原生 RenderNode 内容
        → 像素合成到 Compose Canvas
```

### ExternalRenderNode 两种创建路径

| 路径 | 方法 | 适用场景 | 触发条件 |
|------|------|---------|---------|
| CAPI 路径 | `makeForCApi(rootNodeId, extNodeId, frameNode)` | CAPI RenderNode（新设备） | `enableCApi=true` 且 `harmonyApiVersion >= 22` |
| ArkTS 路径 | `makeForArkTs(renderNode)` | ArkTS RenderNode（回退） | `enableCApi=false` |

- CAPI 路径通过 `rootNodeId` 和 `extNodeId` 定位 FrameNode，获取 `OH_ArkUI_RenderNodeUtils_GetRenderNode`
- ArkTS 路径直接使用 ArkTS RenderNode 对象

### InteropViewPainter 绘制基类

`InteropViewPainter` 是同层渲染的绘制基类，持有 `ExternalRenderNode` 引用：

```kotlin
abstract class InteropViewPainter(srcOffset: IntOffset) : Painter() {
    protected lateinit var externalRenderNode: ExternalRenderNode
    var srcSize: IntSize = IntSize.Zero

    override fun DrawScope.onDraw() {
        drawContext.canvas.nativeCanvas.drawRenderNode(
            externalRenderNode,
            SkRect.makeWH(size.width, size.height)
        )
    }
}
```

子类 `ArkUIViewPainter` 和 `ArkUINativeViewPainter` 分别对应两种原生视图类型。

### 策略路由：useStacked() 决策

`FusionInteropViewStrategy` 通过 `useStacked()` 判断使用叠层还是同层：

```kotlin
private fun useStacked(): Boolean {
    val container = LocalArkUIViewController.current as? ComposeArkUIViewContainer ?: return true
    val enableCApi = container.fusionState?.enableCApi ?: false
    return enableCApi && container.harmonyApiVersion < 22
}
```

| 条件 | 结果 | 混排方式 |
|------|------|---------|
| 非 FusionRenderer 路径 | `true`（叠层） | 三明治结构 |
| `enableCApi && apiVersion < 22` | `true`（叠层） | 旧设备回退 |
| `enableCApi && apiVersion >= 22` | `false`（同层） | ExternalRenderNode |
| `!enableCApi`（ArkTS RenderNode） | `false`（同层） | ExternalRenderNode |

### ArkUIView 同层实现

`ArkUIViewFusionRenderNode` 的关键流程：

1. **创建 ExternalRenderNode**：根据 `enableCApi` 选择 CAPI 或 ArkTS 路径
2. **创建 Painter**：`ArkUIViewPainter` 包装 ExternalRenderNode
3. **布局**：`PlaceInteropLayout` 执行测量和定位
4. **Modifier 链**：`.paint(painter).trackUIKitInteropRenderNode().pointerInteropFilterRenderNode()`
   - `paint(painter)` — 将 ExternalRenderNode 绘制到 Compose Canvas
   - `trackUIKitInteropRenderNode()` — 追踪 RenderNode 位置变化
   - `pointerInteropFilterRenderNode()` — 处理触摸事件坐标映射

### ArkUINativeView 同层实现

`ArkUINativeViewFusionRenderNode` 用于 CAPI 原生视图：

1. **挂载**：`ExternalRenderNode.mountHandleToNodeContent(nodeContent, handle)` 将原生视图挂到 NodeContent
2. **坐标映射**：通过 `bindNativeViewMapFrameNodePos()` 和 `externalRenderNode.mapPointFromGlobal(x, y)` 将触摸坐标从全局坐标系转换到原生视图坐标系

### 触摸事件坐标映射

同层渲染中触摸事件需要跨坐标系映射：

```
Compose 触摸事件 (x, y)
  → ExternalRenderNode.mapPointFromGlobal(x, y)
    → 原生组件坐标系 (px, py)
```

ArkTS 侧通过 `mapFrameNodePos` JS 函数建立桥梁，Kotlin 侧调用 `externalRenderNode.mapPointFromGlobal()` 完成转换。

### C++ 层实现

`RenderNodeImplC.cpp` 中：
- 通过 `OH_ArkUI_RenderNodeUtils_GetRenderNode` 获取 ExternalRenderNode 句柄
- 通过 `OH_ArkUI_RenderNodeUtils_AddChild` 将其添加为 RenderNode 的子节点

## 关键要点

- 同层渲染仅存在于 FusionRenderer 路径；SkiaRenderer 路径始终使用三明治叠层
- `useStacked()` 的决策依据是设备 API 版本，`harmonyApiVersion >= 22` 时启用同层
- ExternalRenderNode 是 skiko 提供的桥接对象，封装了 OHOS 原生 RenderNode 的跨语言访问
- 同层渲染消除了挖洞需求，原生组件直接参与 Compose 的绘制遍历
- 触摸事件通过 `mapPointFromGlobal()` 进行坐标映射，确保命中测试正确
