# 三明治混排结构

## 定义

OHOS 自渲染路径中 Compose UI 与 ArkUI 原生组件混合布局的设计方案，通过 Stack 叠放多层 NodeContainer 实现原生组件在 Compose 内容的上方或下方显示。

## 详解

### 五层叠放

```
TextToolbarOverlay (第5层：文本工具栏 + PasteButton)
NodeContainer(touchableRootView) (第4层：触摸事件拦截)
NodeContainer(foreRootView) (第3层：前景原生组件)
XComponent(TEXTURE) (第2层：Compose 渲染面)
NodeContainer(backRootView) (第1层：背景原生组件)
```

### InteropContainer 三层选择

| 值 | 视觉位置 | 挖洞 | 典型场景 |
|---|---------|------|---------|
| BACK（默认） | Compose 下面 | 自动挖透明洞 | 大部分原生组件 |
| FORE | Compose 上面 | 不需要 | 安全组件、输入法 |
| TOUCHABLE | 最顶层 | 不需要 | 仅拦截事件的透明层 |

### BACK 层挖洞机制

`drawRect(Color.Transparent, blendMode = BlendMode.DstAtop)` 在原生组件的精确位置将 Compose 像素擦除为透明，让下面的原生组件可见。

### 默认 BACK 的设计原因

BACK 让原生组件成为 Compose UI 树的"一等公民"——能被覆盖、能被遮挡、Z 序由布局决定。FORE 是"钉在最上层"的例外情况。

## 关键要点

- 三个 ArkUIRootView 是独立的 NodeController，Kotlin 层按需添加节点
- 挖洞精确限定在组件位置和尺寸范围内
- PasteButton 使用 OHOS 安全组件机制，用户点击即授权，无需 manifest 权限
- 文本工具栏通过 Messenger JSON 通道控制显示/隐藏

## 局限性

三明治混排结构仅适用于 **SkiaRenderer（自渲染）路径**。FusionRenderer 路径从 `harmonyApiVersion >= 22` 起使用 [[同层渲染]]，通过 `ExternalRenderNode` 直接将原生组件嵌入 Compose 绘制管线，无需叠层和挖洞。
