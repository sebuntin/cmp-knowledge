# 手势事件处理机制

## 定义

CMP 中触摸事件从 OHOS 系统到 Compose 手势识别的六阶段跨语言传递与处理机制。

## 详解

### 六阶段链路

| 阶段 | 位置 | 操作 |
|------|------|------|
| 1. ArkTS 捕获 | `.onTouch()` 回调 | OHOS TouchEvent → 拖拽代理更新 + dispatchTouchEvent |
| 2. 坐标变换 | ArkUIView（仅互操作视图） | vp→px 转换 + 父容器偏移校正 |
| 3. NAPI 桥接 | arkui_view_controller.cpp | napi_value 跨语言传递到 Kotlin |
| 4. 策略分发 | InputStrategy | 检查是否应该处理事件 |
| 5. 事件转换 | ComposeSceneMediator | TouchType→PointerEventType、坐标×density、多指追踪 |
| 6. Compose 处理 | PointerInputEventProcessor | 命中测试 → 事件分发 → 手势识别 |

### 两种渲染路径的差异

| 维度 | Fusion Renderer | SkiaRender |
|------|----------------|------------|
| 触摸来源 | NodeContainer.onTouch | Stack.onTouch |
| 就绪判断 | container.isActive() | isEglReady() |
| 未就绪返回 | true（不消费） | false（拒绝） |
| EGL 依赖 | 无 | 必须 |

### 事件转换核心（ComposeSceneMediator）

- **类型映射**：Down→Press, Up→Release, Move→Move, Cancel→特殊处理
- **坐标缩放**：原始坐标 × density → Compose 逻辑坐标
- **多指追踪**：activeChangedPointers 字典维护所有活跃触控点
- **Cancel 处理**：清空所有指针 + 通知 Compose 取消进行中手势

### 关键优化

- **GC 抑制**：Press 时抑制 KN 垃圾回收，避免触摸卡顿
- **历史触摸点**：高频采样数据附加到每个 Pointer，提升速度计算精度

## 关键要点

- 事件转换发生在 ComposeSceneMediator，是 OHOS 特有逻辑
- 策略分发是渲染模式隔离的体现，InputStrategy 接口解耦了判断条件
- Compose 内部处理（阶段6）是平台通用逻辑，与 Android 一致
