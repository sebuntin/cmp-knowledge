# LLM Wiki 使用指南

本指南说明如何使用这个基于 Karpathy LLM Wiki 模式构建的个人知识库系统。

## 系统架构一览

```
cmp-knowledge/
├── CLAUDE.md          ← 📋 Schema（LLM 行为规范）
├── raw/               ← 📥 原始源文档（不可变）
│   └── assets/        ← 🖼️ 图片资源
├── wiki/              ← 📚 LLM 生成的知识 Wiki
│   ├── index.md       ← 📑 内容目录
│   ├── log.md         ← 📝 活动日志
│   ├── concepts/      ← 💡 概念页（核心概念、技术术语）
│   ├── entities/      ← 📦 实体页（类、组件、模块）
│   ├── sources/       ← 📄 源文档摘要页（src- 前缀）
│   └── analysis/      ← 📊 分析页（对比、深度分析）
└── (现有文档可留在原位)
```

## 三步开始使用

### 第一步：添加源文档

把你想学习的材料放入 `raw/` 目录：

- 用 Obsidian Web Clipper 剪藏网页文章
- 复制/移动 PDF、Markdown、文本文件
- 截图放入 `raw/assets/`
- 你 vault 中已有的文档（如 `CMP融合渲染架构设计文档.md`）也可以作为源

**源文档是不可变的** —— LLM 只读取，绝不修改你的原始文件。

### 第二步：告诉 LLM 来处理

打开 Claudian（就是你现在使用的 AI 助手），说：

> **"请录入 raw/xxx.md"** 或 **"请处理 CMP融合渲染架构设计文档.md"**

LLM 会：
1. 阅读源文档
2. 与你讨论关键发现
3. 创建摘要页、概念页、实体页
4. 更新交叉引用
5. 更新 [[index]] 和 [[log]]

### 第三步：浏览和查询

- **浏览**：在 Obsidian 中打开 [[index]] 查看所有页面
- **提问**：对 LLM 说 **"Wiki 中关于 XXX 的内容是什么？"**
- **图谱**：用 Obsidian 的 Graph View 可视化知识网络
- **追问**：深入探索，LLM 会基于已有知识综合回答

## 日常操作速查

| 你想做什么    | 怎么说                                |
| -------- | ---------------------------------- |
| 录入新文档    | "请录入 raw/xxx.md"                   |
| 处理已有文档   | "请把 CMP融合渲染架构设计文档.md 录入 Wiki"      |
| 提问       | "Wiki 中关于 Fusion Renderer 的内容是什么？" |
| 对比分析     | "对比两种渲染路径的帧驱动机制"                   |
| 健康检查     | "请 Lint Wiki，检查是否有矛盾或孤儿页面"         |
| Web 搜索补充 | "帮我搜索关于 XXX 的最新信息并录入 Wiki"         |
| 创建特定页面   | "帮我创建一个关于 XXX 的概念页"                |

## 最佳实践

1. **一次录入一个源文档**，保持参与感，检查 LLM 的摘要质量
2. **有价值的问答也保存下来** —— 如果 LLM 给出了好的分析或对比，要求它创建新的分析页
3. **定期 Lint** —— 每录入 5-10 个源文档后，要求做一次健康检查
4. **善用 Graph View** —— 在 Obsidian 中查看知识网络图谱，发现意想不到的关联
5. **用标签分类** —— 在 frontmatter 中使用 tags 字段，便于 Dataview 查询

## 工作流示例

### 场景：学习一个新技术

```
你: 把这篇论文 raw/paper-xxx.md 录入 Wiki

Claudian:
  我读完了这篇论文，以下是关键发现：
  1. ...
  2. ...
  要特别关注哪些方面？

你: 重点看第三章的算法部分

Claudian:
  好的，我创建了以下页面：
  - wiki/src-paper-xxx.md （源文档摘要）
  - wiki/某算法.md （概念页）
  - 更新了 wiki/index.md
  你可以在 Obsidian 中查看。
```

### 场景：对比分析

```
你: Wiki 中关于两种渲染路径的区别是什么？

Claudian:
  基于已有知识，两种路径的主要区别是...
  [引用多个 Wiki 页面]
  要我把这个对比保存为分析页吗？

你: 好

Claudian:
  已创建 wiki/analysis-渲染路径对比.md
```

## 数据安全

- `raw/` 中的原始文件永远不会被修改
- `wiki/` 中的内容由 LLM 生成，你可以自由编辑
- 整个 vault 可以用 Git 版本控制，所有变更可追溯
- 当 Wiki 规模增长后，`index.md` + Obsidian 搜索足以应对数百页的规模
